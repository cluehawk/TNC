## Page 3 — Overview, Objective, Loss Conditions

## Overview
You awake to discover that you and several others are imprisoned within a pitch-black labyrinth. The rough-hewn stone passages barely allow movement. Equipped only with dim candles, the group must work together to explore the maze and escape. Candlelight only illuminates immediate surroundings, and something moves in the darkness beyond the flicker.

## Object
**The Night Cage** is a cooperative tile-placement labyrinth game for **1 to 5 players**.  
Each prisoner has only a candle. The light reveals adjacent passages only. As players move, **new pathways are revealed** while **old ones disappear into darkness**.

## Win Condition
To win:
1. **Each Prisoner must collect a Key.**
2. The group must **find a single Gate**.
3. All Prisoners must **assemble together on that Gate**.
4. The Gate is unlocked only when the Keys are turned **in unison**.

## Loss Conditions
The Prisoners lose if:
- **All 4 Gate Tiles have been lost.**
- The Prisoners **cannot each collect a Key**.
- The Prisoners **cannot all bring their Keys to a Gate** before darkness isolates them.

## Page 5 — Setup (Player Count Rules, Prisoner Materials)

## Prisoner Materials
Each Prisoner gets:
- matching Prisoner Status card
- matching candle-shaped Prisoner token
- **1 Nerve token**
- **1 Starting tile**

## 1–4 Player Games
Use the **6x6** side of the Game Board.

Remove **2 Keys** from the tile pool and return them to the box.

Remaining play pool:
- **6 Keys**
- **12 Wax Eaters**

### 1 Player Game
- The player controls **all four Prisoners**, taking a turn for each.

### 3 Player Game
- Each player controls one Prisoner.
- The **4th Prisoner** is a dummy/shared player.
- Control begins with the host/owner using the **4th Prisoner marker**.
- When the 4th Prisoner acts, the current holder takes that turn.
- Afterward the marker passes clockwise.

### 4 Player Game
- Each player controls one Prisoner.

## 5 Player Games
Use the **7x7** side of the Game Board.

Remove from the tile pool:
- **1 Key**
- **2 Wax Eaters**

Remaining play pool:
- **7 Keys**
- **10 Wax Eaters**

### 2 Player Game
- Each player controls **two Prisoners**.
- Recommended: take both of your Prisoners’ turns consecutively rather than alternating.

## Tile Holder Assembly
1. Insert the candle sides into the slots on the candle back.
2. Lit side faces outward.
3. Slot the floor into the side pieces.

## Page 6 — Setup (Board, Opening Tiles, Draw Stack, Discard Board)

## Table Setup
- Place the Game Board in the center.
- Place the metal Key tokens on the corner of the board.
- Return unused tiles to the box:
  - extra Starting tiles
  - removed player-count tiles
  - Advanced Game tiles (if not used)

## Opening Tile Stack
For **1–4 players**, set aside:
- 4 T tiles
- 2 four-way tiles
- 2 straight tiles

For **5 players**, set aside:
- 5 T tiles
- 3 four-way tiles
- 2 straight tiles

Shuffle these face down as the **opening tiles**.

## Build the Draw Stack
1. Mix all remaining tiles face down.
2. Load them randomly into the Tile Holder.
3. Put the opening stack on top.

The tiles in the holder are called the **Draw Stack**.  
They represent the remaining candlelight and hope of escape.

### Important
When all tiles have been drawn, you can no longer illuminate new passages.  
See **Final Flickers**.

## Discard Board
Use the Discard Board with the correct player-count side up.

Discarded tiles are sorted by type to:
- track Gates, Keys, and Monsters encountered
- support strategy
- tell whether the game is already lost
- simplify resorting for the next game

## First Player
- Owner/host goes first.
- Play proceeds clockwise.
- In a 3-player game, give the shared **4th Prisoner Marker** to that player to start.

## Page 7 — Starting Positions, First Turn Setup, Board Wrap

## On Each Prisoner’s First Turn
1. Choose any **unoccupied** board space.
2. Place that Prisoner’s **Starting tile** there in any orientation.
3. Place the Prisoner token on the Starting tile.
4. Draw and place tiles one at a time from the Tile Holder:
   - flip each tile face up to inspect it
   - choose a space that the Starting tile exits onto
   - place the tile so passages connect
   - otherwise orient it freely

After all Prisoners complete this, normal play begins.

## Board Wrap: A Cage Without Bars
The board edges wrap to the opposite side in an infinite loop.

Rules:
- Prisoners can move from one edge to the opposite side.
- When at an edge, illuminate relevant opposite-edge spaces too.
- Monsters also attack across wrapping edges if connected by passage.

## FAQ Notes
### Can a Starting tile connect to another Prisoner’s corridor?
Yes. Starting near other players can reduce tile draws, but may increase monster danger.

### Can a Starting tile be placed next to a dead end?
Yes. It becomes an impassable wall. If a lit candle still illuminates that wall, the tile stays in place.

### Can a Prisoner start completely dead-ended?
Yes. But it is likely a disadvantage. The Prisoner will be unable to move and will **Fall through a Pit** on their first turn (see Pits).

## Page 8 — Light and Darkness

## Candlelight Radius
A Prisoner’s candle illuminates **connected passages in a 1-space radius** around the current location.
- **No diagonals**
- Only connected passages are illuminated

## Movement and Darkness
When a Prisoner moves:
- the radius of candlelight moves too
- new passages are revealed
- tiles too far outside the candlelight are **lost to darkness**
- those tiles are discarded forever

This means backtracking does **not** restore the same maze; new passages may appear.

## Important Adjacent Tile Rule
Adjacent tiles not connected by a passage should also be discarded.

**Exception:**  
Adjacent tiles that block a passage remain, because the Prisoner can still see the wall they create.

## Page 9 — Turn Structure: Move vs Stay

## Core Turn Rule
On each turn, the active Prisoner must choose:
- **MOVE**
- **STAY**

Strategy discussion is encouraged in multiplayer, but the controlling player makes the final decision.

## Staying
When a Prisoner chooses **Stay**:
1. Gain **1 Nerve**.
2. Discard **1 tile** from the top of the Draw Stack face up.
3. Place the discarded tile on the Discard Board.

### Monster Exception While Staying
If the discarded tile is a **Monster**:
- do **not** discard it
- instead, you **must replace one connected tile** with that Monster tile

### Additional Staying Notes
- Staying does **not** trigger Monster attacks.
- If you Stay on a **Crumbling tile**, you **Fall** when it collapses that turn.
- Falling can trigger Monster attacks.
- Then the turn ends.

## Moving
When a Prisoner chooses **Move**:
1. Move **1 space** along a connected passage onto a connected tile.
2. Passages are too narrow for more than one Prisoner on the same tile.

### Gate Exception
- Any number of Prisoners may occupy a **Gate tile** at the same time.

### After Moving
1. Remove all tiles no longer illuminated and discard them.
2. Illuminate new passages.
3. Draw and place tiles one at a time:
   - draw 1 tile for each open passage from the new space
   - inspect it
   - choose which open space to place it in
4. No tile is drawn for passages already illuminated by another Prisoner.
5. Tiles may be rotated as long as the passageway connects to the current tile.
6. Continue until all passages from the current location are illuminated.
7. If the Draw Stack is empty, skip illumination.
8. Remove all tiles no longer illuminated and discard them.
9. End turn and pass play clockwise.

## Page 10 — Crumbling Tiles, Pits, Falling

## Crumbling Tiles
The following tiles show thin crack lines and are **Crumbling**:
- Starting tiles
- Straight Passages
- 2 of the T tiles
- Key tiles

A Crumbling tile collapses into a **Pit** the turn **after** you step on it.

### If You Move off a Crumbling Tile
- It collapses as you scramble forward.
- Flip it over to reveal the Pit.

### If You Stay on a Crumbling Tile
- It collapses beneath you.
- You **Fall**.

Pits remain until they are no longer illuminated, then are removed normally.

## Falling
Falling is not always death. It can even be useful.

A Prisoner may fall by:
- having a Pit form beneath them
- moving into an adjacent Pit
- leaping into a Pit intentionally if connected

When a Prisoner Falls:
- the turn ends immediately
- the Prisoner falls through darkness off the board

### Falling Procedure
1. Place the Prisoner token along the board edge, aligned with the row or column once occupied.
2. Discard any tiles no longer illuminated by that Prisoner’s light.
3. At the start of the next turn:
   - choose an empty, unlit space along the selected row or column
   - draw a tile to land on
   - place the Prisoner token there
   - illuminate connected spaces
4. Then take the turn as normal.

### Falling Dangers
- If the landing tile is a **Monster**, it attacks immediately before further illumination.
- After the attack, the Prisoner scrambles onto an existing adjacent tile, or draws a new tile into an unlit space and moves there.
- If there are **no tiles left in the Draw Stack** to land on, the Prisoner falls forever and the game is lost.
- If there are no empty spaces, land on an existing tile.

## Page 11 — Key Tiles and Gate Tiles

## Key Tiles
Key tiles are drawn and placed like normal tiles.

To claim a Key:
- move onto the Key tile
- take a metal Key token from the reserve
- place it by the Prisoner Status card
- this is a free action

## Carry Limit
A Prisoner may carry only **one Key at a time**, because the other hand holds the candle.

## Important Key Risks
If a Prisoner already has a Key and reveals another Key tile:
- walking away may let it be lost to darkness
- Key tiles are Crumbling, so walking through one may cause the Key to be lost when it becomes a Pit
- if the only passage is blocked, no one else may reach it

## Passing Keys
A Prisoner may pass a held Key to another Prisoner if:
- they are adjacent
- they are connected by a passage
- it is one of their turns

Passing a Key is a **free action**.

## Gate Tiles
Gate tiles are drawn and placed like any other tile.

Rules:
- Gate is the only tile that may hold **multiple Prisoners**
- To win, all Prisoners must gather on the **same Gate**
- all must have Keys in hand

If all Gates are lost to darkness, the game is lost.

## Page 12 — Monsters: Wax Eaters, Line of Sight, Attack Triggers

## Wax Eaters
Wax Eaters are the standard monsters in the core game.

They are:
- drawn and placed like any other tile
- motion-sensitive
- hostile to movement in their line of sight

## Trigger Condition
A Wax Eater attacks any time a Prisoner **moves along, into, or out of** a pathway connected to it **in a straight line**.

They attack:
- in all 4 directions simultaneously
- as far as connected uninterrupted pathways continue
- including across board-wrap edges

## Attack Interruption
A Wax Eater cannot see through:
- empty spaces without tiles
- Pits
- interrupted passage connections

## Trigger Sources
Monsters are triggered by:
- a Prisoner moving in line of sight
- a Prisoner falling onto one
- chain reaction when struck by another Monster’s attack

## Evasion by Side Movement
If a Prisoner moves out of line of attack like taking a side passage, the Monster is triggered but the Prisoner may avoid the hit.

Leaping/falling into a Pit can also evade the attack.

---

## Page 13 — Monsters: Chain Reactions, Falling on a Monster

## Chain Reactions
If a Wax Eater’s attack hits another Monster, that Monster also attacks.

This can chain across multiple Monsters.

## Falling onto a Monster
If a falling Prisoner lands on a Monster:
1. The Monster attacks immediately.
2. It hits the falling Prisoner and any other Prisoners in line of sight.
3. After the attack, the falling Prisoner scrambles to:
   - an existing adjacent space, or
   - a newly drawn adjacent tile

## Page 14 — Monster Penalties, Lights Out, Relighting

## Monster Attack Penalties
When struck by a Wax Eater:
- Discard the top **3 tiles** from the Draw Stack face up
- That Prisoner goes **LIGHTS OUT**

These penalties apply:
- once per Prisoner hit
- a Prisoner can be hit by multiple Monsters
- multiple Prisoners can be hit by one attack

This can discard many tiles and cause severe losses.

## Lights Out
When a Prisoner goes Lights Out:
- flip the Prisoner Status card to its Lights Out side
- the candle is extinguished
- the Prisoner can no longer perceive beyond the tile occupied
- remove all tiles previously illuminated by that Prisoner, unless still lit by another Prisoner

### Lights Out Penalties
- The Prisoner is panicked and **must Move every turn**
- They may **not Stay**, unless spending Nerve
- When they Move:
  - draw a tile only for the space moved into
  - discard the tile left behind if no one else illuminates it
  - the new tile must connect to the previous location
- If they move onto a Monster:
  - it attacks immediately
  - they remain Lights Out
  - discard 3 tiles
  - scramble onto an existing adjacent tile or a newly drawn one

## Relighting
A Lights Out Prisoner is relit when:
- they become adjacent to a Prisoner with a lit candle
- the two are connected by a passage

This may happen on either Prisoner’s turn.

When relit:
- the current player draws tiles to fill all newly illuminated passages at the end of that turn
- flip the Status card back to Lit

## Page 15 — Nerve and Final Flickers

## Nerve
All Prisoners begin the game with **1 Nerve token**.

Additional Nerve:
- gained by choosing **Stay**
- maximum **2 Nerve**

One or both Nerve may be used on a turn.

## Spend 1 Nerve To
- **Move Again** — may spend one or both Nerve this way
- **Stay** — while Lights Out, when otherwise forbidden
- **Block** — discard only 2 tiles instead of 3 when hit by a Monster
- **Charge** — deliberately move onto a Monster tile, immediately suffering its attack and going Lights Out, then move onto an existing or newly drawn tile
- **Sustain** — during Final Flickers, skip removing an illuminated tile after your turn

## Final Flickers
When the **Draw Stack is empty**:
- no new tiles can be added to the board
- play continues using only tiles still on the board

At the end of every Prisoner’s turn:
1. remove tiles no longer illuminated
2. also remove **one additional illuminated tile from anywhere on the board**
3. unless Nerve is spent to prevent this

A Monster may be removed this way.

### Important
During Final Flickers, the additional tile is removed **only after all extra Moves** made through spending Nerve.

## Additional Loss Notes
The players lose if:
- darkness isolates a Prisoner so they cannot bring their Key to the Gate
- a falling Prisoner cannot draw a landing tile and falls forever
- too many Keys or Gates are lost

## Page 16 — Advanced Game: Keepers

## Advanced Game Setup Changes
Remove from the standard pool:
- all Key tiles
- 8 Wax Eaters, leaving 4

Add:
- **6 Keepers** for 1–4 players
- **7 Keepers** for 5 players
- **2 Pit Fiends**

Continue setup otherwise normally.

## Keepers
Keepers replace Key tiles in the Advanced Game.

Properties:
- Keys are inside them
- they remain on the board even when not illuminated
- they stay until their Key is removed

### Placement and Facing
- The **Mouth** of the Keeper always faces the player placing it

### Attack Behavior
- Triggered like Wax Eaters
- Attack in **3 directions**
- Cannot attack behind themselves
- Ranged hit forces discard of **1 tile**
- Keeper ranged attack **cannot be Blocked** with Nerve
- If hit from behind by another monster, no chain reaction occurs

### Taking a Key from a Keeper
- Moving onto a Keeper earns a Key just like a Key tile
- Approaching from the rear is safe
- Moving or Falling onto a Keeper from a non-rear direction causes:
  - Lights Out
  - discard 3 tiles
- This 3-tile penalty **can** be Blocked with Nerve
- Once the Key is collected, the Keeper becomes a Pit
- A falling Prisoner without a Key may take the Key when landing there

## Page 17 — Advanced Game: Pit Fiends

## Pit Fiends
Pit Fiends do not attack Prisoners directly.  
Instead they transform tiles into Pits.

### Behavior
- Placed like a normal tile
- Immediately turn every tile in a **diagonal line** from the Pit Fiend into Pits
- This includes:
  - Key tiles
  - Gate tiles
  - Monster tiles
- Empty spaces do not stop the diagonal effect

After attacking:
- the Pit Fiend itself becomes a Pit

### Limits
- Pit Fiend effects **do not wrap**
- they stop at board edges

### Prisoner Interaction
- Any Prisoner standing on a tile turned into a Pit falls immediately, out of turn
- then normal turn order resumes

Pit Fiends are especially dangerous late in the game because they can remove last Gates or critical Keys.

Optional harder mode:
- add a 3rd Pit Fiend

## Page 18 — Boss Monsters: The Dirge and The Pathless

## Boss Monsters
Optional challenge additions:
- **The Pathless**
- **The Dirge**

They are designed to create dread and force new strategies.

## Difficulty Adjustment
To increase difficulty:
- remove a Key or Gate from recommended counts
- or remove both

To make the game easier:
- add 1 or 2 extra Key tiles

## The Dirge
A colossal Monster occupying **9 spaces**.

### Omens
Shuffle **2 Omen tiles** into the Draw Stack.

- When the **1st Omen** is revealed:
  - place it beside the board
  - redraw a replacement tile
- When the **2nd Omen** is revealed:
  - place it on the board as normal
  - this signals the Dirge’s arrival
  - it acts as the anchor point for Dirge placement

### Placing the Dirge
- The Dirge must overlap the Omen tile
- Orientation/placement is chosen by the placer
- It must fit entirely within board boundaries
- No part may extend off the board

### Effect
Everything beneath the Dirge is swallowed in darkness.
Any Prisoners beneath it:
- go Lights Out
- discard 3 tiles
- then fall into a Pit

The attack can be Blocked with Nerve, reducing discard to 2.

After resolving:
- all tiles under the Dirge are discarded
- flip the Dirge to its Pit side
- it becomes a 9-space Pit
- the Pit stays until no longer illuminated

## The Pathless
There are **3 Pathless** tiles total.

To add it:
- shuffle **2 Pathless** tiles into the Draw Stack
- add the third for harder difficulty

### Pathless Rules
Triggered like a Wax Eater, but:
- attack is **not prevented by walls**
- not prevented by **gaps**
- not prevented by **Pits**
- attacks every space in the row and column it intersects

Penalty:
- discard **5 tiles**
- cannot be Blocked with Nerve

## Page 20 — Turn Progression Summary

## Turn Progression
Play proceeds clockwise to the next Prisoner.

### If Falling
- choose an unlit space to land
- draw and place a tile
- place the Prisoner on it
- illuminate all connected passages

### Pits Form
If you:
- **Stayed on** a Crumbling tile, or
- **Moved from** a Crumbling tile

then it flips and becomes a Pit.

### Move
- Move 1 space.

### Stay
- Gain 1 Nerve.
- Discard 1 tile from the Draw Stack.
- If the discarded tile is a Monster, swap it with a connected tile.

### Trigger Monsters
If you moved or fell while in a Monster’s line of sight, trigger that Monster.

### Light and Darkness
- Remove and discard tiles no longer illuminated.
- Draw, orient, and place newly illuminated tiles.
- If a Lights Out Prisoner was relit this turn, the current player draws their newly illuminated tiles.

### Hit by Monsters
Any Prisoner still in line of sight of a triggered Monster:
- discard 3 tiles
- go Lights Out

### Final Flickers
Discard one additional illuminated tile from anywhere on the board.

Optional:
- spend Nerve to Move Again, then return to the Move step

### Falling
If the tile you are on is a Pit, Fall immediately.

## 8088 / Text-Mode Adaptation Notes
These are not original rules, just engineering-friendly extraction notes:
- Treat the board as a fixed wraparound grid
- Represent tile topology explicitly: N/E/S/W openings
- Track illumination ownership by Prisoner or lit-state source count
- Support disappearing tiles when illumination drops to zero unless still visible as blocking walls
- Gate is the only multi-occupancy tile
- Monsters require line tracing through connected passages, including wrap logic
- Lights Out reduces visibility to current tile / movement destination logic
- Falling requires storing a pending re-entry row or column

## Win / Loss Evaluation Summary
### Win
- every Prisoner has a Key
- all Prisoners occupy the same Gate simultaneously

### Loss
- all Gates lost
- insufficient Keys remain obtainable for all Prisoners
- a Prisoner can no longer reach a Gate with the group
- a falling Prisoner cannot land because Draw Stack is exhausted
- endgame isolation prevents group escape
